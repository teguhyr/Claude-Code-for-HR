# Level 2: Intermediate — Claude Code for HR Professionals

> **Important:** All content generated by Claude Code is a draft starting point. Have qualified HR and legal professionals review any documents before use in your organization.

**Prerequisites:** Complete Level 1 (`01-basic-hr-claude-code.md`). You should be comfortable saving output to files and writing basic HR prompts.

---

## Level 2 Goals

By the end of this level, you will be able to:

- Map the full employee lifecycle and apply Claude Code at every stage
- Chain multiple prompts together to build complete HR document sets
- Create end-to-end workflows for recruitment, onboarding, performance, and offboarding
- Build reusable workflow "recipes" that you can run again and again
- Use Claude Code for learning and development planning

**New Claude Code techniques at this level:**
- Multi-turn interactive sessions (back-and-forth refinement)
- Prompt chaining (output from one prompt feeds into the next)
- Workflow recipe files (plain-English instructions saved in `workflows/`)

**HR concepts at this level:**
- Dave Ulrich's **Employee Champion** role — advocating for employees through structured, fair, and growth-oriented HR processes
- The full **Employee Lifecycle** — from attraction to offboarding
- Preview of Josh Bersin's **Employee Experience** (EX) — onboarding is not a day, it's a 90-day experience

**Time to complete:** 4–6 hours

## 2. Recruitment Workflow

Good recruitment starts before the job description goes live. Claude Code helps you build a consistent, bias-reduced recruitment process.

### Step 1 — Job description (from Level 1 template)

You already know how to do this. Use the template from Level 1.

### Step 2 — Screening criteria

Once the JD is written, generate the criteria your team will use to screen applications.

```bash
claude -p "Based on this job description, write a structured screening criteria guide for reviewing resumes.
[PASTE YOUR JOB DESCRIPTION]
Include:
- Must-have criteria (deal breakers if missing)
- Nice-to-have criteria
- Red flags to watch for
- Recommended yes/maybe/no scoring guide
Keep it objective and focused on skills and experience, not personal characteristics." > output/screening-criteria-[role].md
```

### Step 3 — Interview question bank

**Goal:** Generate a behavioural interview question set for a specific role.

```bash
claude -p "Write a structured behavioural interview question bank for a [JOB TITLE] role.
Include 5 questions for each of these competencies:
- Problem-solving and analytical thinking
- Communication and stakeholder management
- Teamwork and collaboration
- Adaptability and resilience
- [ROLE-SPECIFIC COMPETENCY]
Use the STAR format reminder (Situation, Task, Action, Result) at the top.
Include 2-3 suggested follow-up probes for each question." > output/interview-questions-[role].md
```

### Step 4 — Interview scoring rubric

```bash
claude -p "Create a structured interview scoring rubric for assessing candidates for a [JOB TITLE] role.
Include:
- A 1-5 rating scale with clear descriptions for each level
- Scoring columns for each competency from the interview question bank
- An overall recommendation section (Strongly recommend / Recommend / Neutral / Do not recommend)
- Interviewer notes section
Format as a table where possible." > output/interview-scoring-rubric-[role].md
```

**Chaining these together:** Run all four steps in sequence. Each output file feeds context into the next prompt. By the end, you have a complete recruitment toolkit for one role.

## 3. Onboarding Workflow (Detailed)

> **Bersin preview:** Josh Bersin's research shows that employees who have a structured 90-day onboarding experience are significantly more likely to stay past 3 years. Onboarding is not orientation — it is a 90-day integration program. We cover this fully in Level 3.

Onboarding has two audiences: the new hire and their manager. Build documents for both.

### 30-day detailed schedule (new hire view)

```bash
claude -p "Create a detailed 30-day onboarding schedule for a new [JOB TITLE] joining a [COMPANY TYPE].
Organize by week (Week 1, Week 2, Week 3, Week 4).
For each week, include:
- Theme/focus for the week
- Daily suggested activities (not hour-by-hour, just key tasks)
- Key people to meet
- Systems or tools to get access to
- End-of-week reflection question for the new hire
Include placeholders for [MANAGER NAME], [BUDDY NAME], [TEAM NAME]." > output/onboarding-30day-schedule-template.md
```

### Manager's onboarding guide

```bash
claude -p "Write a manager's guide to onboarding a new team member effectively.
Cover these phases:
- Before Day 1 (preparing the team, workspace, tools)
- Week 1 (first impressions, immediate priorities)
- First 30 days (structure, check-ins, early wins)
- Days 31-90 (increasing autonomy, feedback loops, integration)
Include practical tips, not just theory.
Add a 'Common mistakes managers make' section at the end.
Use placeholders for [MANAGER NAME] and [NEW HIRE NAME]." > output/manager-onboarding-guide.md
```

### Batch generating an onboarding pack

You can generate multiple onboarding documents in one terminal session using a simple script pattern:

```bash
# Generate welcome email
claude -p "Write a warm welcome email from HR to a new hire starting Monday. Include placeholders for [NAME] and [START DATE]." > output/onboarding-welcome-email.md

# Generate IT setup checklist
claude -p "Create an IT setup checklist for a new employee's first day. Include: laptop setup, email configuration, key software tools, VPN, and security training. Use checkbox format." > output/onboarding-it-checklist.md

# Generate culture overview
claude -p "Write a one-page company culture overview for a new employee welcome pack. Include: how decisions get made, communication norms, meeting culture, and how to ask for help. Use placeholders for [COMPANY NAME]." > output/onboarding-culture-overview.md
```

Each command runs independently and saves to a separate file. You now have a three-document onboarding pack in minutes.

## 4. Performance Management Documents

Performance management is one of the most sensitive and legally consequential areas of HR. Use Claude Code to draft frameworks and templates — always review before use.

### OKR / SMART goal-setting template

```bash
claude -p "Write a goal-setting guide for employees using the SMART framework (Specific, Measurable, Achievable, Relevant, Time-bound).
Include:
- A 2-paragraph explanation of SMART goals in plain language
- 3 example SMART goals for different types of roles (technical, managerial, support)
- A blank template for employees to fill in their own goals
- Common mistakes people make when setting goals, and how to avoid them" > output/smart-goal-setting-guide.md
```

### Mid-year check-in guide

```bash
claude -p "Create a guide for managers on how to run a productive mid-year performance check-in.
Include:
- Purpose of the check-in (not a rating — a conversation)
- Suggested agenda (30-minute format)
- 10 open-ended questions the manager can ask
- How to document the conversation
- What to do if the conversation reveals performance concerns
Keep tone supportive and coaching-focused." > output/mid-year-checkin-guide.md
```

### Performance Improvement Plan (PIP) template

> **Legal reminder:** PIPs carry significant legal risk. This template is a starting point only. Always involve an employment lawyer before issuing a PIP.

```bash
claude -p "Draft a Performance Improvement Plan (PIP) template for an employee with a performance concern.
Include these sections:
- Purpose and intent (this is a support tool, not a punishment)
- Description of performance concern (with placeholder for specifics)
- Performance expectations going forward (SMART format)
- Support and resources provided by the company
- Timeline and check-in schedule (typically 30-90 days)
- Consequences if performance does not improve
- Signatures section (employee, manager, HR)
Write in neutral, factual, non-punitive language.
Include a prominent legal review reminder at the top." > output/pip-template.md
```

## 5. Learning and Development Planning

L&D (Learning and Development) is where HR moves from compliance to growth. Claude Code helps you build learning plans and identify skill gaps quickly.

### Skills gap analysis prompt

```bash
claude -p "Create a skills gap analysis template for a [JOB TITLE] role.
Structure:
- List the top 10 skills required for the role (split: technical skills / soft skills)
- For each skill, include a 1-5 proficiency scale with descriptions
- Add a column for 'Current Level', 'Target Level', and 'Development Action'
- Include a summary section for prioritizing the top 3 development areas
Format as a table." > output/skills-gap-analysis-[role].md
```

### Individual Development Plan (IDP)

```bash
claude -p "Write an Individual Development Plan (IDP) template for a professional employee.
Include:
- Employee details section (name, role, manager, review date — all placeholders)
- Career aspirations (where do they want to be in 2-3 years?)
- Top 3 development goals for the next 12 months
- For each goal: actions, resources needed, timeline, success indicator
- Manager support section (what will the manager do to help?)
- Quarterly check-in prompts
Keep it one page maximum when completed." > output/individual-development-plan-template.md
```

### Training needs assessment

```bash
claude -p "Create a training needs assessment (TNA) framework for a team of 20 people in a [DEPARTMENT].
Include:
- Purpose and how to use this assessment
- A survey of 10 questions employees answer about their own skills and learning needs
- A parallel survey of 5 questions for managers
- How to collate and prioritize findings
- A template for summarizing results into a team learning plan" > output/training-needs-assessment-[dept].md
```

## 6. Offboarding Workflow

Offboarding is as important as onboarding. A poor exit damages your employer brand, risks IP loss, and loses valuable institutional knowledge.

### Comprehensive offboarding checklist

```bash
claude -p "Create a comprehensive offboarding checklist for an employee leaving a company.
Organize by responsible party:
- IT / Systems (access revocation, device return, data handover)
- HR (payroll final run, benefits termination, reference letter)
- Manager (knowledge transfer, handover document, team communication)
- Employee (outstanding tasks, transition notes, exit survey)
Include a timeline column (e.g., '2 weeks before last day', 'last day', 'within 1 week after')
Use checkbox format." > output/offboarding-checklist.md
```

### Exit interview question set

```bash
claude -p "Create an exit interview question guide for HR to use with departing employees.
Include:
- A brief explanation of the exit interview's purpose (voluntary, confidential)
- 15 open-ended questions covering: reasons for leaving, highlights of their time at the company, what could have been better, feedback on their manager and team, would they return or refer others
- Guidance notes for the interviewer on how to handle sensitive responses
- A simple data capture template to identify patterns across multiple exit interviews" > output/exit-interview-guide.md
```

### Knowledge transfer document

```bash
claude -p "Create a knowledge transfer document template for an employee who is leaving.
Include these sections:
- Role overview and key responsibilities
- Active projects and current status
- Key stakeholders and relationships (internal and external)
- Recurring tasks and processes (with step-by-step notes for each)
- Critical systems and where to find information
- 'What I wish I'd known' section — tribal knowledge, workarounds, tips
- Recommended handover timeline
Use placeholders for [EMPLOYEE NAME], [ROLE], [SUCCESSOR/INTERIM CONTACT]." > output/knowledge-transfer-template.md
```

## 7. Chaining Prompts: Building a Workflow Recipe

A **workflow recipe** is a plain-English file in your `workflows/` folder that describes a multi-step HR process. It tells you (or Claude Code) what prompts to run in what order.

This is the `workflows/` folder concept from the project's `Skill.md` file.

### Create your first workflow recipe

```bash
mkdir -p workflows
```

Now create a recipe file for the new hire onboarding workflow:

**`workflows/new-hire-onboarding-workflow.md`** — create this file with the following content:

```
# Workflow: New Hire Onboarding Document Pack

## Purpose
Generate a complete onboarding document pack for a new hire.

## Inputs needed before running
- [ROLE TITLE]
- [START DATE]
- [MANAGER NAME]
- [BUDDY NAME]
- [DEPARTMENT]
- [COMPANY NAME]

## Steps

Step 1: Generate welcome email
  claude -p "Write a warm welcome email from HR to [NAME] joining as [ROLE TITLE] on [START DATE]. Include manager [MANAGER NAME] as a contact. Tone: warm and human." > output/onboarding-welcome-email.md

Step 2: Generate first-week checklist
  claude -p "Create a Day 1-5 onboarding checklist for [ROLE TITLE] in [DEPARTMENT]. Include IT setup, team introductions, key meetings, and system access. Manager is [MANAGER NAME], buddy is [BUDDY NAME]." > output/onboarding-week1-checklist.md

Step 3: Generate 30-60-90 plan
  claude -p "Create a 30-60-90 day onboarding plan for a new [ROLE TITLE] at [COMPANY NAME]. For each phase include: focus theme, 3-4 milestones, success indicators." > output/onboarding-30-60-90-plan.md

Step 4: Generate manager guide
  claude -p "Write a guide for [MANAGER NAME] on how to onboard a new [ROLE TITLE] effectively. Cover: before day 1, week 1, first 30 days, days 31-90. Include practical manager tips." > output/onboarding-manager-guide.md

## Output files
- output/onboarding-welcome-email.md
- output/onboarding-week1-checklist.md
- output/onboarding-30-60-90-plan.md
- output/onboarding-manager-guide.md
```

**Why this matters:** You've just turned a four-step prompt sequence into a reusable recipe. Next time a new hire joins, you open this file, fill in the placeholders, and run all four steps in order. That's HR-as-Code in practice.

### Exercise A: Full Recruitment Pipeline

**Goal:** Build a complete recruitment document set for one role using chained prompts.

**Steps:**
1. Pick a role (e.g., HR Business Partner, Finance Manager, Software Engineer)
2. Run four prompts in sequence:
   - Step 1: Job description → `output/jd-[role].md`
   - Step 2: Screening criteria (using JD as context) → `output/screening-[role].md`
   - Step 3: Interview question bank (referencing the competencies in the JD) → `output/interview-questions-[role].md`
   - Step 4: Scoring rubric → `output/scoring-rubric-[role].md`

**Expected output:** Four linked documents that form a complete recruitment toolkit for that role.

**Where to save:** `output/` folder with consistent naming.

### Exercise B: 90-Day Onboarding Plan (Detailed)

**Goal:** Create a complete, role-specific 90-day onboarding program.

**Prompt to use:**
```bash
claude -p "Create a detailed 90-day onboarding plan for a new [ROLE TITLE] at a [COMPANY TYPE].
For each phase (Days 1-30, Days 31-60, Days 61-90):
- Primary focus/theme
- Week-by-week objectives (brief bullet points)
- Key meetings and people to engage
- Learning milestones
- Success indicators for the phase
- Suggested manager check-in questions
Include a 'Warning Signs' section: indicators that onboarding is not going well and what HR/managers should do.
Use placeholders for [MANAGER NAME], [HR CONTACT], [COMPANY NAME]." > output/90-day-onboarding-plan-[role].md
```

**Where to save:** `output/90-day-onboarding-plan-[role].md`
