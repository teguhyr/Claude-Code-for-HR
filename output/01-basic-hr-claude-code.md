## 1. Your First HR Prompt

Let's start simple. Open your terminal, navigate to your project folder, and run an interactive session.

```bash
cd /path/to/Claude-Code-for-HR
claude
```

You're now in an interactive session. Claude will greet you and wait for input.

**Exercise — Welcome Email:**

Type this prompt (use `Shift+Enter` to go to a new line, then `Enter` to submit):

```
Write a welcome email for a new employee joining our company on Monday.
The email is from the HR team. Keep it warm, professional, and under 150 words.
Include a placeholder for the employee's name and start date.
```

**What to do with the output:**
- Copy it to a new file, OR
- Exit the session (`Ctrl+C`) and run the same prompt using the save-to-file method:

```bash
claude -p "Write a welcome email for a new employee joining our company on Monday. The email is from the HR team. Keep it warm, professional, and under 150 words. Include a placeholder for the employee's name and start date." > output/welcome-email-template.md
```

**Key habit:** Every time you generate something useful, save it to `output/` immediately.

## 2. Writing Job Descriptions

Job descriptions are one of the most time-consuming HR writing tasks. Claude Code makes them fast and consistent.

### Why job descriptions matter

- Legally, they define the role and can be used in performance management and termination decisions
- For candidates, they are your first impression — clarity attracts better applicants
- Internally, they align hiring managers and HR on what the role actually requires

### Basic job description prompt template

```bash
claude -p "Write a job description for a [JOB TITLE] at a [COMPANY TYPE].
Include these sections:
- Role summary (2-3 sentences)
- Key responsibilities (6-8 bullet points)
- Required qualifications (4-5 bullet points)
- Preferred qualifications (3-4 bullet points)
- What we offer (3-4 bullet points)
Keep language clear, direct, and inclusive." > output/jd-[job-title].md
```

**Replace `[JOB TITLE]` and `[COMPANY TYPE]` before running.**

Example — try this now:

```bash
claude -p "Write a job description for a People Operations Coordinator at a 200-person SaaS startup. Include these sections: Role summary (2-3 sentences), Key responsibilities (6-8 bullet points), Required qualifications (4-5 bullet points), Preferred qualifications (3-4 bullet points), What we offer (3-4 bullet points). Keep language clear, direct, and inclusive." > output/jd-people-ops-coordinator.md
```

### Auditing for gender-coded language

Research shows that certain words in job descriptions discourage candidates from applying based on gender. Ask Claude Code to check your drafts.

```bash
claude -p "Review this job description for gender-coded language and suggest more neutral alternatives. [PASTE YOUR JD TEXT HERE]"
```

Or if you've already saved the file, start an interactive session and paste the content:

```
claude
> [paste your job description]
> Now check this for any gender-coded words like 'aggressive', 'dominant', 'nurturing', or 'supportive' that might discourage certain candidates. Suggest neutral replacements.
```

## 3. Drafting HR Policies

HR policies establish the rules and expectations for how the organization operates. They need to be clear, consistent, and legally sound.

> **Legal reminder:** Policies have legal implications. Always have an employment lawyer or qualified HR leader review before distributing.

### Types of policies to practice with

- Remote work / hybrid work policy
- Code of conduct (excerpt)
- Dress code policy
- Leave and time-off policy
- Anti-harassment policy (summary/overview only — this one especially needs legal review)

### Drafting a remote work policy

```bash
claude -p "Draft a remote work policy for a 100-person technology company.
Use these sections:
- Purpose
- Eligibility
- Work hours and availability expectations
- Equipment and home office setup
- Data security requirements
- Manager responsibilities
- Review and updates
Write in plain, clear language. Use bullet points within each section." > output/remote-work-policy-draft.md
```

### Using placeholders for reusability

Good policy templates use placeholders so you can reuse them across organizations or update them easily. Ask Claude Code to add them:

```bash
claude -p "Rewrite this policy draft with placeholders for company-specific details like [COMPANY NAME], [EFFECTIVE DATE], [HR CONTACT EMAIL], and [MANAGER TITLE]. Keep all other text the same." > output/remote-work-policy-template.md
```

### Prompt tip: Specify reading level

If your workforce is non-technical or multilingual:

```
Add this to any policy prompt: "Write at a Grade 8 reading level. Avoid HR jargon."
```

## 4. Onboarding Welcome Documents

Onboarding is the first impression a new employee gets of your organization. Research consistently shows that structured onboarding improves retention and time-to-productivity.

> **Bersin preview:** Josh Bersin's research identifies onboarding as one of the highest-impact "moments that matter" in the employee lifecycle. We cover this fully in Level 3.

### New hire first-week checklist

```bash
claude -p "Create a first-week checklist for a new employee joining a mid-sized company.
Organize by day (Day 1, Day 2, Day 3, Day 4, Day 5).
Include tasks for: IT setup, team introductions, key system access, culture orientation, role-specific onboarding.
Use checkboxes (- [ ] format).
Include placeholders for [MANAGER NAME], [BUDDY/MENTOR NAME], and [DEPARTMENT]." > output/new-hire-first-week-checklist.md
```

### CEO/leadership welcome message

```bash
claude -p "Write a welcome message from a CEO to a new employee joining the company.
Tone: warm, genuine, not corporate-speak.
Length: 3 short paragraphs.
Cover: excitement about the new hire joining, what makes the company's culture special, an open-door message.
Include placeholders for [EMPLOYEE NAME], [CEO NAME], and [COMPANY NAME]." > output/ceo-welcome-message-template.md
```

### Culture and values overview

```bash
claude -p "Write a one-page 'Who We Are' culture overview for a new employee welcome pack.
Include sections: Our Mission, How We Work, What We Value, How We Make Decisions.
Keep it human and conversational — not a list of corporate buzzwords.
Use placeholders for [COMPANY NAME] and [FOUNDING YEAR]." > output/culture-overview-template.md
```

## 5. Basic Troubleshooting

### Output is too long

Add length instructions to your prompt:
```
"Keep this under 300 words." or "Use no more than 5 bullet points per section."
```

### Output is too generic or vague

Add more context about your organization:
```
"This is for a 50-person healthcare startup in Australia. Our team is mostly remote."
```

### Claude keeps repeating itself

Start a fresh session:
```bash
/clear
```
Or exit and restart: `Ctrl+C`, then `claude` again.

### The output cut off mid-way

This usually means the response was too long for one generation. In an interactive session, type:
```
Continue from where you left off.
```

### Rate limit error

If you see a rate limit error, wait 60 seconds and try again. For large batches of documents, space your requests out.

### The output doesn't sound like my company

Claude Code doesn't know your organization. Give it context:

```
"Our company values are: [VALUE 1], [VALUE 2], [VALUE 3]. Our tone is [FORMAL/CASUAL]. We are a [INDUSTRY] company with [NUMBER] employees."
```

### Exercise A: Offer Letter Template

**Goal:** Create a reusable offer letter template for professional roles.

**Prompt to use:**
```bash
claude -p "Draft a formal employment offer letter template for a full-time professional role.
Include these sections:
- Offer of employment (role title, department, start date)
- Compensation (base salary, pay frequency)
- Benefits summary (health, retirement, PTO — use placeholder values)
- Employment conditions (at-will employment, background check, references)
- Next steps and acceptance deadline
Use placeholders for all company-specific and candidate-specific details.
Write in a professional but welcoming tone." > output/offer-letter-template.md
```

**Expected output:** A complete offer letter with clear sections and placeholders like `[CANDIDATE NAME]`, `[ROLE TITLE]`, `[START DATE]`, `[SALARY]`, `[COMPANY NAME]`.

**Where to save:** `output/offer-letter-template.md`

### Exercise B: 30-60-90 Day Onboarding Plan

**Goal:** Create a structured onboarding plan outline for a new hire.

**Prompt to use:**
```bash
claude -p "Create a 30-60-90 day onboarding plan for a new mid-level employee joining a company.
For each phase (Days 1-30, Days 31-60, Days 61-90), include:
- Primary focus/theme for the phase
- Key milestones (3-4 bullet points)
- Success indicators (how you know the phase went well)
- Manager responsibilities during this phase
Use placeholders for [ROLE TITLE], [DEPARTMENT], and [MANAGER NAME].
Keep it practical and actionable." > output/30-60-90-onboarding-plan-template.md
```

**Expected output:** A three-phase onboarding roadmap with clear goals and checkpoints for each phase.

**Where to save:** `output/30-60-90-onboarding-plan-template.md`

### Exercise C: Disciplinary Warning Letter Template

**Goal:** Create a legally cautious template for a first written warning.

**Prompt to use:**
```bash
claude -p "Draft a first written warning letter template for an employee performance issue.
Include these sections:
- Purpose of the letter
- Description of the performance concern (with placeholder for specific details)
- Previous conversations or informal warnings (placeholder)
- Expected behavior going forward (specific, measurable)
- Consequences if behavior does not improve
- Employee acknowledgment section
Write in a neutral, factual tone. Avoid accusatory language.
Include a note at the top reminding HR to have legal review before use.
Use placeholders for all specific details." > output/disciplinary-warning-letter-template.md
```

**Expected output:** A structured warning letter template that is factual, neutral, and legally cautious.

**Where to save:** `output/disciplinary-warning-letter-template.md`
