# Claude Code for HR Professionals — Overview

> **Important:** This learning path teaches you how to use Claude Code as a productivity tool for HR work. All documents generated by Claude Code are draft starting points. Always have qualified HR and legal professionals review any content before use in your organization.

---

## What is Claude Code?

Claude Code is Anthropic's official command-line tool (CLI) that lets you interact with the Claude AI model directly from your terminal or command prompt — without opening a browser.

**Why does this matter for HR?**

- Most HR teams copy-paste work into a web chat interface, one task at a time
- Claude Code lets you run HR tasks from your own computer, save output directly to files, automate repetitive work, and build a personal library of reusable HR templates
- It turns HR document work into a repeatable, version-controlled process — this is the "HR-as-Code" idea at the heart of this project

**In plain English:** Instead of typing into a web browser, you type into your terminal. The output goes directly into files you own, organized how you want.

## Installation

### What you need first

- A computer running macOS, Linux, or Windows (with WSL or Git Bash)
- Node.js version 18 or higher — check with: `node --version`
- npm (comes with Node.js) — check with: `npm --version`
- An Anthropic API key OR a Claude Max subscription

> **Not sure if you have Node.js?** Go to [nodejs.org](https://nodejs.org) and download the LTS version. It includes npm automatically.

### Step-by-step installation

**Step 1 — Install Claude Code globally:**
```bash
npm install -g @anthropic-ai/claude-code
```

**Step 2 — Authenticate:**
```bash
claude
```
The first time you run this, Claude Code will prompt you to log in. Follow the on-screen steps to connect your Anthropic account or paste your API key.

**Step 3 — Verify the install worked:**
```bash
claude --version
```
You should see a version number like `1.x.x`. If you do, you're ready.

> **Billing note:** Claude Code uses your Anthropic account. If you have a Claude Max subscription, usage is included. If you're on API billing, you pay per token. For learning purposes, usage costs are minimal.

## Core Commands Reference

These are the commands you will use throughout this learning path.

| Command | What it does | HR use case example |
|---|---|---|
| `claude` | Opens an interactive session (like a chat) | Have a back-and-forth conversation to refine a job description |
| `claude "your prompt"` | Runs a single prompt and exits | Quick one-off tasks: `claude "Write a subject line for an offer email"` |
| `claude -p "your prompt"` | Runs a prompt in non-interactive mode, outputs plain text | Pipe output into a file or another command |
| `claude -p "prompt" > output/file.md` | Runs a prompt and saves the result directly to a file | Generate a policy draft and save it instantly |
| `/help` | Shows available commands (use inside a session) | Remind yourself what's available |
| `/clear` | Clears the current conversation context | Start fresh after a long session |
| `/model` | Switch the AI model mid-session | Switch to a faster or more powerful model |
| `Ctrl + C` | Cancels the current generation | Stop Claude if it's generating something you don't need |
| `Shift + Enter` | Adds a new line without submitting (inside a session) | Write multi-paragraph prompts |

### Saving output to files

This is one of the most useful habits to build. Instead of copy-pasting from a browser:

```bash
claude -p "Draft a remote work policy for a 50-person company." > output/remote-work-policy.md
```

This saves the output directly to `output/remote-work-policy.md` in your current folder. All exercises in this learning path use the `output/` folder.

## The HR-as-Code Philosophy

Traditional HR work looks like this: open a browser, write a prompt, read the output, copy it, paste it into Word, edit it, save it somewhere.

The HR-as-Code approach looks like this:

1. **Write once, reuse many times** — Store your best prompts as templates in `workflows/`
2. **Output goes straight to files** — No copy-paste; documents land where they belong
3. **Track changes over time** — Because files are text, you can use git to see who changed what and when (covered in Level 3)
4. **Build a library** — Over time, your `output/` and `workflows/` folders become an organizational knowledge base

**Two frameworks anchor the advanced level of this course:**

- **Dave Ulrich's HR Model** — Defines the four strategic roles HR must play in an organization: Strategic Partner, Administrative Expert, Employee Champion, and Change Agent. This model helps HR move from reactive administration to proactive business partnership.

- **Josh Bersin's Employee Experience (EX) Approach** — Frames every HR initiative around how it affects the employee's daily life. Instead of asking "did we run the program?", EX asks "did the employee's life actually get better?" The three pillars are Productivity, Wellbeing, and Meaning.

Both frameworks are introduced in Level 1, grow in Level 2, and become the organizing structure of Level 3.

## Quick Start: Your First Command

Once Claude Code is installed, try this right now:

```bash
claude -p "Write a two-sentence welcome message for a new HR team member joining a company for the first time." > output/first-output.md
```

Open `output/first-output.md`. You should see a short, professional welcome message. That's Claude Code working.
